# 방송자료 디지털화 영상 검사 프로그램

2026년 방송자료 디지털화 지원 사업의 감리에 쓰려고 만든 영상 품질 검사 프로그램입니다.
사업수행사가 납품한 디지털 변환 영상을 표본으로 뽑아 재생 여부, 파일 규격, 화질과 음질,
파일명과 납품목록의 정합성을 자동으로 점검합니다. 점검 결과는 감리수행결과보고서에 그대로
붙일 수 있는 형태로 정리합니다.

검사 엔진은 방송·아카이브 분야에서 사실상 표준으로 쓰는 FFmpeg입니다. 프로그램은 이 FFmpeg를
안에 담은 실행파일 하나로 배포하므로, 검수 장소의 PC에 따로 설치할 프로그램이 없습니다.

판정은 감리원이 의견을 낼 때 쓰는 근거 자료입니다. 최종 판정은 감리원이 원본과 대조해 확정합니다.


## 필요성

디지털화 대상은 약 40,200시간, 메타데이터 고도화 대상은 약 40,400시간에 이릅니다. 이 분량을
사람이 눈으로만 확인하기는 어렵고, 검사하는 사람에 따라 판정이 갈리거나 빠뜨리는 부분이 생깁니다.

제안요청서도 실제 구축자료를 표본으로 뽑아 실증적으로 검증하고, 표본검수 방식과 선정 기준을
감리계획서에 밝히도록 요구합니다. 이 프로그램은 그 요구에 맞춰 표본 검증을 자동으로 돌리고,
판정 근거를 수치와 시각(타임코드)으로 남깁니다.


## 검사 범위

납품 영상 전체를 모집단으로 두고 표본을 뽑아 검사합니다. 보존용 마스터(MXF·MOV 등 저압축·무손실)와
활용·서비스용 열람본(MP4, H.264)을 파일 경로 규칙으로 자동 구분해 각각의 기준으로 봅니다. 여기에
납품목록(엑셀·CSV), 폴더 구조, 파일과 자료ID의 연결까지 대조합니다.

감리 중점 점검 항목과 검사 영역은 아래와 같이 맞물립니다.

| 감리 점검 항목(제안요청서) | 검사 영역 |
|---|---|
| 변환 파일의 재생 여부, 저장매체 이상 | 무결성(전체 디코드) |
| 변환 파일 규격 준수 | 컨테이너·코덱·해상도·프레임레이트·스캔·비트레이트 |
| 화질 | 블랙·정지·레터박스·휘도/채도 이상 |
| 음질 | 무음·라우드니스·트루피크 |
| 파일명·폴더구조·납품목록 정합성 | 명명규칙·금지문자·경로·목록 대조 |
| 중복·누락, 원본–파일–메타 연결 | 납품목록 교차대조 |
| 사업수행사 자체 품질검수 적정성 | 자동 검사 결과와 자체검수 결과 비교 |

자동 검사로 가리지 못하는 부분도 분명히 해 둡니다. 자료 제목이나 출연자, 방영일 같은 내용 확인,
색보정이 잘 됐는지에 대한 판단, 화면 속 개인정보 판독은 프로그램이 다루지 않습니다. 이런 항목은
감리원이 직접 보고 담당자 면담으로 확인합니다. 자동 검사에서 주의나 미달로 나온 파일은 원본과
대조해 한 번 더 확인한 뒤 의견으로 굳힙니다.


## 검사 방법

검사는 여섯 단계로 이어집니다. 납품 폴더를 훑어 영상 목록을 만들고, 통계 기준으로 표본을 뽑고,
파일마다 아래 항목을 순서대로 돌린 뒤, 항목별로 판정하고 증적을 남기고, 리포트를 만들고, 끝으로
감리원이 표본을 눈으로 확인해 판정을 확정합니다.

| 검사 영역 | 세부 검사 | 사용 기법 | 잡아내는 결함 |
|---|---|---|---|
| 무결성 | 전체 스트림 디코드 | 디코드 오류 집계 | 파일 손상, 불완전 변환, 재생 불가 |
| 기술규격 | 컨테이너·코덱·해상도·프레임레이트·스캔·비트레이트·오디오 | 메타데이터 대조 | 규격 위반, 과압축, 인터레이스 미제거 |
| 화질 | 블랙 프레임 | blackdetect | 무영상 구간(신호 유실) |
| 화질 | 정지 프레임 | freezedetect | 캡처 멈춤, 프레임 드롭 |
| 화질 | 레터·필러박스 | cropdetect | 종횡비 변환 오류 |
| 화질 | 휘도·채도 이상 | signalstats | 신호 레벨 이탈, 색 소실 |
| 음질 | 무음 구간 | silencedetect | 오디오 유실, 싱크 이상 |
| 음질 | 라우드니스·트루피크 | ebur128, astats | 방송 레벨 부적정, 클리핑 |
| 파일명 | 명명규칙·금지문자·확장자·경로 길이 | 규칙 대조 | 자동 처리 오류, 매체 이관 실패 |

항목마다 다섯 등급으로 판정합니다. 기준을 넘기면 통과(PASS), 경계선이면 주의(WARN), 기준에 못 미치면
미달(FAIL), 파일이 열리지 않으면 검사 불가(ERROR), 해당 없으면 생략(SKIP)입니다. 주의는 개선을 권고하고,
미달은 시정조치 대상이 되며, 검사 불가는 재생 불가로 보고 재변환을 요구합니다.

결함은 발생 지점과 수치를 함께 기록합니다. 검사 결과는 다음과 같은 형태로 나옵니다.

```
[FAIL] KBS_B0002_20260102_MST.mxf
       블랙(무영상) 08:00.0–13:00.0  누적 5.0초(23.8%)  → 신호 유실 의심, 재변환 검토
       무음 구간            누적 5.0초(23.8%)  → 오디오 유실·싱크 이상 의심
[FAIL] 납품목록 정합성   누락 1건 · 미등재 2건 · 재생시간 불일치 1건
```

검사 결과는 세 가지로 나옵니다. 엑셀 파일(.xlsx)은 영상 파일을 한 행씩 놓고 무결성·기술규격·화질·
음질·파일명의 판정을 색으로 구분해, 어느 파일의 어느 부분이 문제인지 한눈에 보게 합니다. 요약 외에
항목별 상세, 납품목록 대조, 표본 정보 시트를 함께 담아 감리조서에 그대로 붙일 수 있습니다. 이와 별도로
브라우저로 열어 보는 HTML, 원자료로 보관하는 JSON을 함께 만듭니다. 감리원은 이 결과를 근거로 표본을
눈으로 확인하고 원본과 대조해, 문제점과 개선 권고, 시정조치를 감리수행결과보고서에 반영합니다.

![검사 결과 엑셀 요약 시트](assets/vqc_report_excel.png)

검사 결과 엑셀의 요약 시트입니다. 영상 파일 한 행씩 무결성·기술규격·화질·음질·파일명 판정을 색으로
구분하고, 오른쪽 끝에 주요 지적을 한 줄로 모읍니다. 통과는 초록, 주의는 노랑, 미달은 빨강입니다.

사업수행사가 낸 자체 품질검수 결과가 있으면 이 프로그램의 결과와 맞대어, 수행사 검수가 빠뜨리거나
느슨하게 넘어간 부분이 없는지도 함께 봅니다.


## 표본 선정 기준

제안요청서는 표본검수 방식과 선정 기준을 감리계획서에 밝히도록 요구합니다. 표본은 다음 기준으로
정합니다.

표본 크기는 Cochran 공식에 유한모집단 보정을 적용해 신뢰수준 95%, 허용오차 ±5%로 계산합니다.
여기에 정보시스템 감리기준의 DB 검수 8% 이상 조건을 하한으로 둡니다. 최종 표본은 통계로 계산한
수와 8% 비율로 계산한 수 가운데 큰 쪽을 쓰고, 모집단이 30건 이하로 작으면 전수 검사합니다. 예를
들어 모집단이 1,200개면 통계 표본 292개와 8% 표본 96개 가운데 292개를 택합니다.

표본은 방송사와 매체·프로파일을 기준으로 층을 나눠 고르게 배분하고, 층마다 최소 표본 수를 보장해
특정 방송사나 매체의 결함이 통째로 빠지지 않게 합니다. 표본을 뽑는 방식은 고정 시드 기반이라, 같은
모집단과 기준이면 언제 돌려도 같은 표본이 나옵니다. 발주기관이 그대로 재현해 확인할 수 있다는
뜻입니다.

표본에서 미달 비율이 정해 둔 값을 넘으면 해당 층의 표본을 늘리거나 전수 검사로 바꾸고, 발주기관과
협의해 사업수행사에 전량 재검수나 재변환을 요구합니다.


## 배포와 보안

방송자료 원본은 외부로 내보낼 수 없고 검수는 폐쇄망에서 이뤄집니다. 이 조건에 맞춰 프로그램을
다음과 같이 배포합니다.

검수 장소의 PC에는 실행파일 하나만 들여놓으면 됩니다. FFmpeg를 실행파일 안에 담았으므로 대상
PC에 FFmpeg나 별도 런타임을 설치할 필요가 없습니다. 반입 승인과 보안 검토도 파일 한 개 단위로
끝나, 여러 파일과 설치 프로그램을 들여올 때 생기는 심사 부담이 없습니다.

실행하는 동안 외부와 통신하지 않습니다. 모든 처리는 그 자리에서 끝나며, 영상 원본이나 변환 파일,
메타데이터를 밖으로 내보내거나 복제하지 않고 리포트만 만듭니다. 검사 기준은 실행파일과 별도의 설정
파일로 두어, 사업수행사의 납품 규격이 확정되면 프로그램을 다시 배포하지 않고 기준만 고쳐 씁니다.


## 제안요청서 요구사항과의 대응

| 제안요청서 요구 | 대응 |
|---|---|
| 실제 구축자료를 표본으로 실증 검증 | 통계 표본 + 실측 검사 |
| 표본검수 방식·선정 기준을 감리계획서에 명시 | Cochran·층화·8% 하한, 재현 가능한 선정 |
| 변환 파일 재생 여부, 저장매체 이상 | 전체 디코드 무결성 검사 |
| 화질·음질 | 블랙·정지·휘도·무음·라우드니스 검출 |
| 파일명·폴더구조·납품목록 정합성 | 명명규칙 점검 + 목록 교차대조 |
| 원본–파일 연결, 중복·누락 | 누락·미등재·중복·재생시간 대조 |
| DB 검수 8% 이상 샘플링 | 표본 하한을 8%로 강제 |
| 자체 품질검수 적정성 | 자동 결과와 자체검수 결과 비교 |
| 원본 보안·반출 금지, 폐쇄망 | 무설치 실행파일, 외부 통신 없음 |


## 기대 효과

같은 자료를 다시 검사해도 결과가 같기 때문에, 지적 사항을 두고 근거를 다툴 일이 줄어듭니다.
사람이 일일이 열어 보기 어려운 분량을 표본으로 검증하고 납품목록은 전수로 맞춰 보므로, 놓치는
파일이 줄어듭니다. 단순 반복 검수를 프로그램이 처리하는 만큼, 감리원은 원본 대조나 내용 판단처럼
사람이 봐야 하는 일에 시간을 씁니다. 지적마다 어느 지점에서 무슨 수치가 나왔는지 남으므로,
시정조치를 요구할 때 근거가 뚜렷합니다.

적용합니다. 검사 기준은 설정 파일로 관리하므로 감리 과정에서 그대로 공개할 수 있습니다.
